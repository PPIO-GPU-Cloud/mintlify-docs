## 何时应选择使用 GPU 实例？

GPU 作为一种计算芯片，具有以下优势，非常适合需求高并行度的应用程序：

- 实时高速、强并行计算能力和强浮点计算能力。
- 适用于使用数千个线程的工作负载。
- 在图形处理中，面对大量计算要求，每个任务虽小，但通过管道执行的操作集合的吞吐量远比单个操作的延迟重要。

## GPU 与 CPU 的区别是什么？

- **GPU** 拥有大量专注于处理大规模并发算术运算的算术运算单元（ALU）和相对简单的逻辑控制单元，适合于需要多线程并行的高吞吐量计算场景。
- **CPU** 拥有强大的算术运算单元（ALU）和复杂的逻辑控制单元，适合于对响应速度要求较高且逻辑复杂的串行计算场景。



## 如何查看 GPU 实例的价格？

您可以在价格计算器页面查看 GPU 实例的价格及其配置项（系统盘、数据盘、公网 IP 等）费用。请注意，算力市场显示的为参考价格，具体以控制台实际下单结果为准。

## 购买 GPU 实例后如何使用 GPU 资源？

购买 GPU 实例后，操作系统需安装相应 NVIDIA 驱动以使用 GPU 资源。安装驱动的具体步骤，请参考快速入门 NVIDIA 驱动介绍。

## 系统盘大小是多大的？

目前，系统盘大小为 100 GB。

## 如何查看进程占用显卡的情况？
由于实例均为 Docker 容器，因容器 PID 隔离限制，无法使用 ```nvidia-smi``` 命令查看进程。
您可以安装 ```py3nvml``` 库，通过 Python 代码查看进程。具体安装及使用方法，请参考 [py3nvml 库](https://py3nvml.readthedocs.io/en/latest/)。

```
pip install py3nvml 
py3smi
```
